# 📱 استودیو آرسام | Arsam Studio

یک وب‌سایت کاملاً واکنش‌گرا (Responsive)، راست‌چین (RTL) و بومی با قلم فوق‌العاده زیبای **وزیرمتن (Vazirmatn)** جهت معرفی و بارگیری آسان و امن برنامه‌های اندرویدی متصل به میزبانی ابری گوگل درایو.

این پروژه تحت فریم‌ورک مدرن و فوق‌العاده سریع **React 19 + TypeScript** به همراه کامپایلر انقلابی **Vite** و همچنین استایل‌دهی مدرن **Tailwind CSS v4** طراحی و پیاده‌سازی شده و کاملاً با ساختارهای استاتیک **GitHub Pages** هماهنگ است.

---

## 🚀 ویژگی‌های کلیدی آرسام استودیو

*   **⚡ رابط کاربری بومی راست‌چین (RTL):** طراحی اختصاصی بر اساس الگوهای خوانایی فارسی‌زبانان و قلم وزیرمتن.
*   **🌓 پشتیبانی کامل از تم تیره و روشن (Dark / Light Mode):** بهینه‌سازی رنگ‌ها برای خوانایی بالا در شب و روز با قابلیت ذخیره‌سازی آخرین انتخاب کاربر.
*   **📱 بخش معرفی مفصل برنامه‌ها:** شامل نمایش افقی اسکرین‌شات‌ها به همراه بزرگنمایی تعاملی، ویژگی‌ها و تاریخچه کامل به‌روزرسانی‌ها.
*   **☁️ سیستم توزیع متصل به گوگل درایو (Google Drive APK Links):** تسهیل در بارگیری فایل‌های نصبی `APK` به صورت مستقیم و با نظارت کیفی و اسکن آنتی‌ویروس بومی گوگل.
*   **🛠️ پنل مدیریت بومی و تولیدکننده هوشمند JSON:** دارای یک فرم تعاملی شکیل فرانت‌اند برای تولید پکیج‌های اطلاعاتی جدید برنامه‌ها بدون دغدغه خطاهای گرامری کاما و نقل‌قول‌ها در ویرایش دستی فایل `apps.json`.
*   **🍃 انیمیشن‌های نرم و تعاملی:** پیاده‌سازی افکت‌های انتقال ملایم با کتابخانه قدرتمند **Framer Motion (motion/react)**.

## 🛠️ نحوه راه‌اندازی پروژه به صورت محلی (Local Development)

برای توسعه یا اجرای لوکال پروژه روی رایانه خود مراحل زیر را دنبال کنید:

۱. ابتدا مطمن شوید نسخه پایدار **Node.js** را بر روی سیستم خود نصب کرده‌اید.
۲. مخزن (Repository) را بر روی حافظه کلون کنید.
۳. در مسیر اصلی پروژه، وابستگی‌ها را با دستور زیر نصب کنید:
   ```bash
   npm install
   ```
۴. با استفاده از دستور زیر، سرور توسعه محلی را اجرا کنید:
   ```bash
   npm run dev
   ```
۵. آدرس نمایش داده شده در ترمینال (معمولاً `http://localhost:3000`) را مرورگر خود باز کرده و از پروژه استفاده کنید.

## 🌐 راهنمای انتشار بر روی گیت‌هاب پیجز (GitHub Pages)

برای استقرار و آنلاین کردن این وب‌سایت استاتیک در دامنه گیت‌هاب خود، می‌توانید از دو روش زیر استفاده نمایید:

### روش اول: استقرار خودکار با استفاده از GitHub Actions (توصیه شده و فوق مدرن)

این روش کاملاً اتوماتیک است و با هر بار زدن دستور `git push` به صورت خودکار تغییرات را کامپایل کرده و روی گیت‌هاب پیجز منتشر می‌کند.

۱. پروژه را به یک مخزن تفکیک‌شده جدید در حساب کاربری گیت‌هاب خود متصل و Push کنید.
۲. در مخزن گیت‌هاب، از تب بالایی به بخش **Settings** بروید.
۳. در ستون سمت چپ به بخش **Pages** رجوع کنید.
۴. در بخش **Build and deployment**، گزینه منوی کشویی **Source** را از حالت *Deploy from a branch* به **GitHub Actions** تغییر دهید.
۵. سپس، یک فایل در ساختار پروژه خود با مسیر زیر ایجاد کنید:
   `.github/workflows/deploy.yml`
۶. محتوای زیر را در آن فایل قرار داده و تغییرات را در گیت‌هاب Push کنید:

```yaml
name: Deploy website to GitHub Pages

on:
  push:
    branches:
      - main # نام شاخه اصلی کدهای شما

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  deploy:
    environment:
      name: github-pages
      url: \${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Repository
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build Application
        run: npm run build

      - name: Setup Pages
        uses: actions/configure-pages@v4

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'

      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

کارهای شما تمام شد! پس از چند ثانیه، فرآیند گیت‌هاب صفحات شما را کامپایل کرده و آدرس رسمی انتشار را در اختیارتان می‌گذارد.

### روش دوم: انتشار سریع به وسیله پکیج `gh-pages`

اگر تمایلی به استفاده از اکشن‌های اتوماتیک ندارید، می‌توانید به صورت دستی اقدام به بیلد و انتشار کنید:

۱. ابتدا بسته دستوری را نصب کنید:
   ```bash
   npm install --save-dev gh-pages
   ```
۲. فایل `package.json` را باز کنید و خط پایینی را تحت شناسه بیرونی اصلی اضافه کنید (به جای `username` نام کاربری گیت‌هاب و `repository-name` نام مخزن خود را بنویسید):
   ```json
   "homepage": "https://username.github.io/repository-name",
   ```
۳. در تگ میانی `"scripts"` دستورات تکمیلی زیر را مستقر کنید:
   ```json
   "predeploy": "npm run build",
   "deploy": "gh-pages -d dist"
   ```
۴. حال کافیست هر زمان که مایل بودید، ترمینال را باز کرده و فرمان زیر را اجرا کنید تا صفحات کامپایل و مستقیماً روی سرور گیت‌هاب آپلود شوند:
   ```bash
   npm run deploy
   ```
